We'll compare two approaches: Vanilla JavaScript vs React

Vanilla JavaScript - HTML Structure

HTML Setup

<!DOCTYPE html>
<html>
<head>
    <title>Vanilla JS Application</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <div id="app"></div>
    <script src="app.js"></script>
</body>
</html>

Simple structure: All logic handled in JavaScript

Note: Everything depends on manual DOM manipulation

Vanilla JavaScript - Implementation

// Manual DOM manipulation
function showUserList(users) {
  let html = "<h2>User List</h2><div class='user-list'>";
  users.forEach(user => {
    html += `
      <div class="user-card">
        <img src="https://ui-avatars.com/api/?name=${user.name}" 
             alt="${user.name}" />
        <h3>${user.name}</h3>
        <p><strong>Address:</strong> ${user.address}</p>
      </div>
    `;
  });
  html += "</div>";
  app.innerHTML = html; // Replace entire content
}

function fetchUsers() {
  isLoading = true;
  showSpinner("Loading users...");
  
  fetch("http://localhost:8004/api/users")
    .then(res => res.json())
    .then(users => {
      setTimeout(() => {
        isLoading = false;
        showUserList(users);
      }, 2000);
    })
    .catch(err => {
      isLoading = false;
      showError(err.message);
    });
}

Issues with this approach:

  • String concatenation for HTML creation
  • Manual state management
  • No component reusability
  • Complex error handling
  • Hard to maintain and debug

React Components

UserCard Component

// Reusable component
function UserCard({ user }: { user: User }) {
  return (
    <div className="user-card">
      <img 
        src={`https://ui-avatars.com/api/?name=${user.name}`}
        alt={user.name} 
      />
      <h3>{user.name}</h3>
      <p><strong>Address:</strong> {user.address}</p>
    </div>
  );
}

UserList Component

function UserList({ users }: { users: User[] }) {
  return (
    <>
      <h2>User List</h2>
      <div className="user-list">
        {users.map(user => (
          <UserCard key={user.id} user={user} />
        ))}
      </div>
    </>
  );
}

Clean structure: Each component has one responsibility

Reusable: Components can be used anywhere

React State Management

Main App Component

function App() {
  const [users, setUsers] = useState<User[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchUsers = async () => {
      try {
        setLoading(true);
        const fetchedUsers = await UserService.getAllUsers();
        await new Promise(resolve => setTimeout(resolve, 2000));
        setUsers(fetchedUsers);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };
    fetchUsers();
  }, []);

  if (loading) return <Spinner />;
  if (error) return <ErrorComponent onRetry={handleRetry} />;
  return <UserList users={users} />;
}

React Advantages:

  • Automatic re-rendering when state changes
  • Clean separation of concerns
  • Easy to test individual components
  • TypeScript support
  • Declarative programming style
